Git + GitHub Basics

Git = your project’s time machine. GitHub = the cloud for that time machine.

First time only: initialize, commit, connect, push.
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin <repo-url>
git push -u origin main

After that, your regular loop is just:

git add .
git commit -m "message"
git push

JavaScript Essentials (for React)

this keyword

Inside an object method, this refers to the current object.

const cart = {
  id: 1,
  showId() { console.log(this.id); }
}
cart.showId(); // 1

Spread operator ...

const numbers = [1,2,3];
const newNumbers = [...numbers, 4]; // [1,2,3,4]

Array destructuring

const [a, b] = [10, 20];
console.log(a); // 10

.map() transforms arrays

[1,2,3].map(n => n * 2); // [2,4,6]

React Basics

  • JSX = HTML + JS. React uses Babel to compile it for the browser.
  • Insert JS with braces: <p>2+2 = {2+2}</p>
  • All tags must close: <img />, <input />
  • Conditional render with &&: 
    {isLoggedIn && <Dashboard />}

Components

Components are functions that return JSX. Start names with a Capital letter.

function ChatInput() {
  return (
    <>
      <input placeholder="Type..." />
      <button>Send</button>
    </>
  );
}
/* Use like HTML: */
<ChatInput />

Props (Component Inputs)

<ChatMessage message="Hello" sender="User" />

function ChatMessage({ message, sender }) {
  return <p>{sender}: {message}</p>
}

Events

Use camelCase like onClick, onChange. Pass a function, not a function call.

<button onClick={sendMessage}>Send</button>  // ✅
<button onClick={sendMessage()}>Send</button> // ❌ runs immediately

State (Dynamic UI Data)

When state changes, React re-renders the UI. Don’t mutate directly; use the setter.

import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);
  return (
    <>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </>
  );
}
Notes: State updates are batched and not immediate. For list updates, make copies: 
setItems(prev => [...prev, newItem])

Lists need keys

{items.map(item => (
  <li key={item.id}>{item.text}</li>
))}

Lifting State Up (Sharing Between Siblings)

function Parent() {
  const [message, setMessage] = useState("");
  return (
    <>
      <Input setMessage={setMessage} />
      <Display message={message} />
    </>
  );
}

Hooks

Rules: Call hooks at the top level of a component. Never inside loops, conditions, or nested functions.

useState

Explained above.

useEffect

import { useEffect } from "react";

useEffect(() => {
  console.log("runs on mount + when dependency changes");
}, [dependency]); // [] runs once on mount

useRef

import { useRef } from "react";

const inputRef = useRef(null);
<input ref={inputRef} />
<button onClick={() => inputRef.current.focus()}>Focus</button>

Best Practices

  • Let React manage the DOM. Prefer state, props, and refs over document.querySelector.
  • Keep components small & reusable.
  • Provide stable key props for list items.
  • Place all hooks at the top of your component.

Styling

  • Inline: <div style={{ color: "red" }} />
  • CSS files: import "./App.css"
  • CSS-in-JS: styled-components, emotion, etc.

⚡ TL;DR

  • React = functions returning JSX.
  • Data flows via props; dynamic state via useState.
  • Events: onClick etc. pass a function.
  • Lists: .map() + unique key.
  • Effects with useEffect; refs with useRef.
  • Share data by lifting state to the parent.
